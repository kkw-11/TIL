## 임베디드 시스템 응용 분야
### 가정 자동화(HA, Home Automation)
* 집에 있는 모든 가전 제품, 전열기구, 현관의 문, 각 방의 창문 등을 자동으로 제어할 수 있는 시스템임
  * 자동 감지 기능 - 가스 누출, 화재, 도난 등
  * 통신 기능 - 환견문이나 창문 등의 상황 안내
  * 원격 제어 기능 - 가전제품과 조명 기기, 음향 기기 등을 켜고 끄는 기능
  * 정보 가전(information appliance) - 가전기기들이 정보를 송신할 수 있도록 네트워크 기능을 장착함(인터넷 냉장고, 디지털TV, 인터넷 세탁기 등)
### 정보 단말기
  * 디지털 기기(휴대폰, PDA, 스마트폰, 영상 정화기 등)
  * 각종 정보 검색, 오락, 메시징, 개니정보관리 등의 복합 기능을 수행함
### 공장 자동화(FA, Factory Automation)
  * 산업 분야에서 생산 공정의 제어에 필요한 시스템
  * 공정을 실시간으로 제어하거나 매우 정밀한 처리가 필요한 공정에서는 고성능 임베디드 시스템을 탑재한 산업용 로봇과 같은 장비가 필요함
### 사무 자동화(OA, Office Automation)
  * 사무 활동의 효율화 및 내용의 직적 향상을 위해 전자 기기등을 사용하여 사무처리를 자동화함(복사기, 프린터, 스캐너, 팩스 등의 기능을 하나로 통합한 복합기 등)
### 물류 분야
  * 바코드를 위한 리더 및 발급기, 무선인식장치, POS(Point of Sales)단말기
  * RFID(근거리에서 무선으로 정보리르 읽을 수 있는 시스템->바코드 대체) 등
### 금융 분야
  * 자동현금입출금기, 신용 카드의 인식기나 발급기, 훼손된 지폐를 검출함
  * 지문 인식, 음성 안내, 감시 카메라를 이용한 감시 기능


## 실시간 시스템
 * 주어진 입력에 대하여 정해진 시간 내에 행동 혹은 반응할 수 없을 때 무넺가 발생하는 시스템
 * 외부 자극에 대하여 어떠한 환경 아래에서도 마감시간 이내에 논리적으로 정확한 출력 혹은 결과를 내는  적시성(timeliness) 가진 시스템
### 실시간 시스템의 구분
 * 마감시간 내에 결과를 얻지 못할 경우 발생하는 문제의 심각성에 따라 다음과 같이 구분함
#### Hard Real Time System
  * 마감 시간을 못 지키면 응용 프로그램이 실패하며 시스템에 치명적인 손실을 줄 수 있음(원자력 발전소 제어, 화재 발생 검축 시스템, 항공기, 우주 왕복선, 자동차, 미사일 제어 등)
  #### Soft Real Time System
  * 마감시간을 못 지킨 지연된 반응은 시스템에 치명적이 아닌 미세한 영향을 줌(데스크탑 컴퓨터, 정보 단말기 등)
     

## 유비쿼터스 컴퓨팅
### 의미
 * 언제 어디서나 어떤 것이라도 온라인 네트워크와 연결되어 서비스를 받는 환경과 공간을 의미함, 공간개념의 변화(물리공간과 전자공간의 융합으로 제3의 공간이 생성됨)
 * xerox사 PARC(Palo Alto Research Center) 연구소의 연구책임자인 마크와이저(mark weiser)소장에 의해 1991년도에 사용됨
 * TRON 프로젝트 연구책임자인 사카무라 켄 교수가 '모든 물건에 컴퓨터'라고 주장한 것이 시작임
 ![image](https://user-images.githubusercontent.com/76929823/120941475-f0287000-c75d-11eb-9a46-ab84dacde3bd.png)
 * IT 혁명은 물리공간을 전자공간속에 심었다면 현재는 컴퓨터를 물리공간 속에 심는 혁명
#### 마크와이저가 언급한 유비쿼터스 컴퓨팅의 특징
* 인간화된 인터페이스(calm technology)로 눈에 보이지 않아야(invisible)함
* 가상공간의 컴퓨팅(virtual realiy)이 아닌 현실 세계의 어느 곳에서나 컴퓨터의 사용(embedded virtuality)이 가능해야 함
* 네트워크에 연결되지 않은 컴퓨터는 유비쿼터스 컴퓨팅이 아님
* 사용자의 상황(장소,시간 날씨 등)에 따라 서비스가 변해야함
#### 차세대 컴퓨팅 기술
* 웨어러블 컴퓨팅(wearable computing)
    * 컴퓨터를 옷이나 안경처럼 착용할 수 있게 함으로써 인테의 일부로 만드는 컴퓨팅 환경
    * 체내 이식형 컴퓨팅(implant computing) 기술로 발전해 나갈 것임
* 퍼베이시브 컴퓨팅(pervasive ocmputing)
    * 개인용 컴퓨터의 핵심 기능들을 다른 종류의 기기에도 적용함
    * 네트워크에 연결된 무수한 기기들로부터 언제 어디서든 접근이 가능함
    * e-비즈니스 까지 수행할 수 있는 컴퓨팅 환경
* 조용한 컴퓨팅(silent ocmputing)
    * 사물에 심어진 컴퓨터들이 주인이 의식하지 않아도 정해진 일을 묵묵히 수행하는 컴퓨팅 환경
* 노매딕 컴퓨팅(nomadic computing)
    * 네트워크의 이동성이 극대화해 특정 장소가 아니라 사용자가 자유자재로 이동하면서 어디서든지 컴퓨터를 사용할 수 이 ㅆ게 하는 기술
    * '항상 연결된(always connnected)' 컴퓨팅 환경
    * 현재는 노트북이나 휴대폰과 같은 기기를 통하여 언제, 어디서든 외부와 접속하며 이동하는 부류를 의미함
* 감지 컴퓨팅(sentient computing)
    * 감지 장치를 통해 정보를 획득하여 사용자에게 정보를 제공하거나 스스로 처리하는 컴퓨팅 패러다임
    * 주변 곳곳에 심어져 있는 센서 네트워크를 통해 그 능력이 더욱 확대됨
* 엑조틱 컴퓨팅(exotic computing)
    * 스스로 생각해 물리공간과 전자공간의 연계를 수행하는 컴퓨팅 환경
    * 집에서 일어나는 모든 상황과 해야할 작업들을 지능적으로 파악하고 실제로 수행함
* 임베디드 컴퓨팅(embedded computing)
    * 컴퓨터가 수행해야할 기능을 미리 프로그래밍하여 사물에 심는 컴퓨팅 환경
* 일회용 컴퓨팅(disposable computing)
    * 모든 사물에 컴퓨터를 심을 수 있도록 컴퓨터를 일회용 종이만큼 저렴하게 만드는 컴퓨팅 환경  

### 유비쿼터스 기반 기술 
* MEMS
    * 초소형 정밀 기계 기술
    * 마이크로 단위의 초소형 부품 및 시스템의 설계와 제작
* RFID
    * 물품에 소형 칩인 RFID 태그를 부착해서 주변 환경 정보를 무선주파수로 전송,처리하는 비접촉식 무선주파수 인식 기술

    * 32비트 주소체계인 IPv4의 단점을 개선하기 위해 개발된 차세대 인터넷 통신 규약
    * 128비트의 주소 체계를 가지므로 거의 무한대의 주소 개수를 가짐
* SOC
    * 각종 반도체를 하나의 칩에 집적한 시스템
* LBS
    * LBS 이동중인 사용자에게 사용자의 위치와 관련된 다양한 정보를 제공하는 서비스

    * 센처, 칩


## 임베디드 시스템 소프트웨어
### 정의 및 특징
#### 정의
 * 임베디드 시스템 소프트웨어 - 하드웨어를 제어하기 위해 임베디드 시스템에 포함되는 명령어 집합
 * 산업 및 군사용 제어기기, 디지털정보가전기기, 자동화센서장비 등에 인공지능을 부야함으로써ㅏ 그 부가가치를 높이기 위한 핵심 소프트웨어
 * 하드웨어와 밀접한 연관이 있기 때문에, 프로세서가 동일하더라도 환경이 조금만 달라지면 동작하지 않음(비용을 줄이기 위해 임베디드 시스템의 하드웨어가 항상 최적화된 부품만 포함하고 있기 때문임)
 * 응용 소프트웨어를 보조기억 장치에서 따로 읽어오지 않고 메모리에서 직접 수행함 
#### 특징 및 활용
* 특징
    * 경량화, 최적화
    * 네트워크 지원
    * 개발도구 필요
    * 틀정시스템 전굥
    * 고신뢰성
    * 밀티미디어 지원
    * 실시간 처리 지원
    * 저전력화
* 활용
    * 스마트서비스
    * 무선게임
    * 무선 인터넷
    * VOD
    * LBS
    * m 커머스
    * 영상전화
### 종류
#### 운영체제
* 작은 크기의 메모리에서도 원활하게 동작하는 것이 바람직함
* 어떤 데이터나 명령이 입력되더라도 응답을 하지 않고 재부팅을 하는 상황은 피해야 함
* 사용자의 입력에 즉각적으로 반응해야 함
* 소형화 및 저가격화가 가능한 임베디드 리눅스가 많이 사용됨
#### 시스템 소프트웨어
* 응용 프로그램을 개발하기 ㅜ이한 시스템 소프트웨어를 필요로 함
#### 응용 소프트웨어
* 운영체제와 시스템 소프트웨어를 이용하여 사용자에게 유익한 작업을 수행하는 소프트웨어를 말함
## 임베디드 시스템 하드웨어
### 개요
#### 프로세서
* 프로그램의 명령어를 처리함
* 임베디드 시스템의 경우 응용 목적에서 필요로 하는 최소 사양의 프로세서를 사용함
#### 메모리
* 프로그램과 데이터를 저장함
* 임베디드 시스템의 경우는 으용 목적에 필요로 하는 최소한의 메모리르 사용함
* 보조기억장치는 예외적인 경우를 제외하고는 임베디드 시스템에서는 사용하지 않음
#### 주변장치
* 시스템과 외부 장치를 연결함
* 임베디드 시스템의 경우는 응용 목적에서 필요로 하는 제한된 주변 장치를 사용함
#### 네트워크장치
* 시스템과 외부장치들을 연결함
* 임베디드 시스템의 경우는 응용 목적에서 필요로 하는 제한된 주변장치를 호용함

## 임베디드 시스템 운영 체제
### 범용 운영체제
* 다양한 프로그램을 지원함
* Linux, Micorosoft Window XP 등의 데으크탑의 운영체제임
* 다양한 장치와 다양한 작업들을 수행함
* 기본적으로 대용량 메모리를 필요로 함
### 임베디드 시스템용 운영체제
* 가격 경쟁력을 갖추어야 함
* 필요에 따라 실시간 처리가 가능해야 함
* 부팅속도가 빠름
* 작은 크기의 메모리에도 원활하게 동작하는 운영체제가 적당함
    * Window CE, Embedded Linux
    * RT-Linux(RealTime-Linux)
    * Embedded system java, personal java
* Window CE
    * MS에서 개발됨
    * 32 비트 윈오우즈와 호환성이 있기 때문에 win32 API를 이용하여 쉽게 프로그램을 포팅할 수 있음
* Embedded Linux
    * 재구성을 통해 크기의 가변성을 이를 수 있음    
    * 공개된 소스코드를 통해 유연성을 확보할 수 있음
* RT-Linux
    * 기존의 커널에 새로이 실시간 커널을 추가함으로써, 최소한의 소스 벼녁ㅇ으로 실시간 기능을 추가함
    * 단점 : 시스템 자원이 충분하지 않으 임베디드용 운영체제로는 덩치가 큼
* Embedded system java, personal java
    * 선 마이크로시스템사에 의해 개발됨
    * 자바의 가장 큰 장점은 어떠한 시스템에서도 Java API와 JVM(Java Virtual Machine)만 있으면 자바코드의 이식이 가능함
    * 임베디드 자바와 퍼스널 자바는 Java API JVM이 내장되어 있어서 ,기존의 사용 RTOS와 연계되어 사용이 가능함
        * RTOS가 포팅된 타겟 시스템에 임베디드 자바나 퍼스널 자바를 연계하여, 그 위에 응용프로그램으로서 자바코드나 자바 애플릿을 실행하는 것이 가능함

## 임베디드 시스템 리눅스
### 운영체제의 개요
#### operating system
* 운영체제: 컴퓨터와 하드웨어와 사용자 간의 인터페이스 역할을 하는 시스템 프로그램
* 컴퓨터 시스템의 각종 자원을 효율적으로 관리,운영 되도록 사용자에게 편리성을 제공함
* 자원 관리자 역할과 응용 소프트웨어에게 컴퓨팅 자원에 대한 서비스 제공자 역할을 함

#### kernel
* 커널: 운영체제를 작동시키는 핵심 프로그램
* 응용 프로그램과 컴퓨터 자원 사이의 인터페이스 역할을 함
* 자원(processor, memory, disk)을 관리하며, 자원을 프로세스에게 적절하게 배분하고, 잘못 사용하는 것을 방지하는 자원관리자 역할을 함
* 커널이 관리하는 컴퓨팅 자원은 물리적 자원과 추상적 자원으로 분류됨
    * 물리적 자원
        * 시스템 구성요소들과 주변장치(프로세서, 메모리, 디스크, 터미널, 네트워크 등)
    * 추상적 자원 : 물리적 자원을 관리하기 위해 운영체제가 추상화 시킨 객체와 물리적 자원에 대응 되지 않는 추상적 객체로 분류
        * 프로세서를 추상화 시킨 객체
            * task와 thread
        * 메모리를 추상화 시킨 객체
            * page, segment
        * 디스크를 추상화시킨 객체
            * file, inode
        * 네트워크를 추상화시킨 객체
            * 통신 프로토콜 및 패킷
        * 물리적 자원에 대응되지 않는 추상적 객체
            * 보안 혹은 사용자 계정에 따른 접근 제어 등
